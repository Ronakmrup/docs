## Overview

  - SDK Connector (native libraries and dependencies)
  - Endpoint URL (for Sanas Cloud connection)
  - Account ID (unique identifier)
  - Account Secret (secure password)
</Info>

## Understanding Your Credentials

Once approved, you'll receive the following via email:

### 1. SDK Connector

**File:** `sanas_remote_sdk_<platform-arch>_<version>.tar.gz`

**File includes:**

- Native Python libraries
- Required dependencies
- Installation script
- Documentation

**How to use:**

```bash
# Extract the SDK connector
tar -zxvf sanas_remote_sdk_<platform-arch>_<version>.tar.gz

# Navigate to directory
cd sanas_remote_sdk_<platform-arch>_<version>

# Run installation
./install.sh
```

### 2. Endpoint URL

**Example**: wss://api-mumbai.sanas.ai or similar

**What it is:**

- Network address to connect to the Sanas Cloud
- Region-specific endpoint for low latency

**How to use:**

```python
init_params.remoteEndpoint = "your_endpoint_url_here"
```

<Note>
  **Self-hosted users:** You won't use this endpoint. Instead, you'll connect to your own infrastructure.
</Note>

### 3. Account ID

**What it is:**

- Unique identifier for your Sanas account
- Used to authenticate SDK requests
- Non-sensitive (but don't share publicly)

**How to use:**

```python
init_params.accountId = "your_account_id_here"
```

### 4. Account Secret

**What it is:**

- Secure password for SDK authentication
- Sensitive credential (keep confidential)
- Used to verify your identity

**How to use:**

```python
init_params.accountSecret = "your_account_secret_here"
```

<Warning>
  **Security Warning:**

  - Never commit the Account Secret to version control
  - Never share in public forums or documentation
  - Rotate immediately if compromised
  - Use environment variables in production
</Warning>

## Complete Authentication Example

Here's how to use your credentials to initialize the SDK:

```python
import sanas_remote_sdk
import os

# Create SDK instance
sdk = sanas_remote_sdk.CreateRemoteSDK()

# Configure initialization parameters
init_params = sanas_remote_sdk.InitParams()

# Set credentials (use environment variables in production)
init_params.remoteEndpoint = os.getenv("SANAS_ENDPOINT", "your_endpoint_url")
init_params.accountId = os.getenv("SANAS_ACCOUNT_ID", "your_account_id")
init_params.accountSecret = os.getenv("SANAS_ACCOUNT_SECRET", "your_account_secret")

# Enable secure media transmission
init_params.secureMedia = True

# Initialize connection
result = sdk.Initialize(init_params)

if result == sanas_remote_sdk.InitResult.SUCCESS:
    print("‚úÖ SDK initialized successfully")
else:
    print(f"‚ùå Initialization failed: {result}")
```

## Environment-Specific Configuration

### Development Environment

```python
# .env.development
SANAS_ENDPOINT=wss://api-mumbai.sanas.ai
SANAS_ACCOUNT_ID=dev-account-id
SANAS_ACCOUNT_SECRET=dev-secret
SANAS_SECURE_MEDIA=false  # Optional for testing
```

### Production Environment

```python
# .env.production
SANAS_ENDPOINT=wss://api-mumbai.sanas.ai
SANAS_ACCOUNT_ID=prod-account-id
SANAS_ACCOUNT_SECRET=prod-secret
SANAS_SECURE_MEDIA=true  # Always true in production
```

### Loading Environment Variables

```python
from dotenv import load_dotenv
import os

# Load environment-specific config
load_dotenv('.env.production')

init_params.remoteEndpoint = os.getenv("SANAS_ENDPOINT")
init_params.accountId = os.getenv("SANAS_ACCOUNT_ID")
init_params.accountSecret = os.getenv("SANAS_ACCOUNT_SECRET")
init_params.secureMedia = os.getenv("SANAS_SECURE_MEDIA", "true").lower() == "true"
```

## Troubleshooting Authentication

<AccordionGroup>
  <Accordion title="Initialization fails with authentication error" icon="triangle-exclamation">
    **Error:** `InitResult.AUTHENTICATION_FAILED`

    **Common causes:**

    1. Incorrect Account ID or Account Secret
    2. Typos in credentials
    3. Copy-paste formatting issues

    **Solutions:**

    ```python
    # Verify credentials (without logging secrets)
    print(f"Endpoint: {init_params.remoteEndpoint}")
    print(f"Account ID: {init_params.accountId}")
    print(f"Secret length: {len(init_params.accountSecret)}")
    
    # Check for hidden characters
    init_params.accountSecret = init_params.accountSecret.strip()
    ```
  </Accordion>
  <Accordion title="Cannot connect to endpoint" icon="triangle-exclamation">
    **Error:** `InitResult.CONNECTION_FAILED`

    **Common causes:**

    1. Network connectivity issues
    2. Firewall blocking outbound connections
    3. Wrong endpoint URL

    **Solutions:**

    - Verify network connectivity: `ping api-mumbai.sanas.ai`
    - Check firewall allows HTTPS/WSS outbound
    - Confirm endpoint URL matches credentials email
    - Try with `secureMedia = False` for testing
  </Accordion>
  <Accordion title="Credentials not received after request" icon="triangle-exclamation">
    **Wait time:** Typically 1-2 business days

    **If delayed:**

    1. Check spam/junk folder
    2. Verify email address in request form
    3. Contact support: [support@sanas.ai](mailto:support@sanas.ai)
  </Accordion>
  <Accordion title="Need credentials for multiple environments" icon="triangle-exclamation">
    **Use case:** Separate dev, staging, production

    **Solution:** Contact support to request additional credential sets for each environment. This allows:

    - Isolated testing environments
    - Separate usage tracking
    - Better security isolation
  </Accordion>
</AccordionGroup>

## Frequently Asked Questions

<AccordionGroup>
  <Accordion title="How long does it take to get credentials?" icon="clock">
    Typically **1-2 business days** after submitting the request form. If you haven't received them within 3 business days, check your spam folder or contact support.
  </Accordion>
  <Accordion title="Can I use the same credentials across multiple applications?" icon="question">
    **Yes**, you can use the same credentials for multiple applications. However, for production, we recommend requesting separate credentials per environment (dev, staging, production) for better isolation and tracking.
  </Accordion>
  <Accordion title="Do credentials expire?" icon="calendar">
    No, credentials do not expire automatically. However, you should rotate them periodically as a security best practice, especially if:

    - A team member with access leaves
    - You suspect credentials may have been compromised
    - Moving from development to production
  </Accordion>
  <Accordion title="What if I lose my credentials?" icon="key">
    Contact [support@sanas.ai](mailto:support@sanas.ai) with your:

    - Account/company name
    - Technical contact information
    - Original request date (if known)

    We'll verify your identity and issue new credentials.
  </Accordion>
  <Accordion title="Can I share credentials with contractors/partners?" icon="users">
    We recommend requesting separate credentials for external parties. This allows:

    - Better security control
    - Separate usage tracking
    - Easy revocation when engagement ends

    Contact support to request additional credential sets.
  </Accordion>
</AccordionGroup>
